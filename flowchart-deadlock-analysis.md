# TQQQ 백테스트 전략 - 전체 플로우차트 및 데드락 분석

> 분석 기준: `strategy.md` (전략 정의) + `backtest.py` (실제 구현)
> 목적: 모든 상태 전이를 도식화하고, 시스템이 영구적으로 갇힐 수 있는 데드락 상태를 식별

---

## 1. 시스템 상태 모델 (State Machine)

### 3가지 주요 모드

```
  +==================+         +=====================+         +==================+
  |   공격 모드       |         |   방어 모드          |         |   초기 포지션     |
  |   (정상 보유)     |         |   (관망 중)          |         |   (initial_pos)  |
  |                  |         |                     |         |                  |
  | trigger_type=""  |         | trigger_type=       |         | initial_pos=True |
  | waiting=False    |         |   "selloff"|"ma200" |         | waiting=False    |
  | tqqq_shares > 0  |         | waiting=True        |         | tqqq_shares > 0  |
  +==================+         +=====================+         +==================+
```

### 방어 모드의 6가지 하위 상태

```
  방어 모드 (waiting=True)
  |
  +-- [S1] selloff 관망, tqqq=0, sell_off_count < 4  (21일 대기)
  |
  +-- [S2] selloff 관망, tqqq=0, sell_off_count >= 4  (42일 타이머)
  |
  +-- [S3] selloff 관망, tqqq>0 (분할매수), sell_off_count < 4   *** 위험 ***
  |
  +-- [S4] selloff 관망, tqqq>0 (분할매수), sell_off_count >= 4  *** 데드락 ***
  |
  +-- [M1] ma200 관망, tqqq=0  (MA5>MA200 streak 대기)
  |
  +-- [M2] ma200 관망, tqqq>0 (분할매수)
```

```
                      |
```

## 3. 이벤트 1: -3% 이벤트 세부 분기 (6가지 케이스)

```
  +=============================================+
  |        -3% 이벤트 발동 (ret <= -0.03)        |
  +=============================================+
                      |
                      v
          +------------------------+
          | initial_pos == True ?   |
          +------------------------+
           |YES                |NO
           v                   v
  +------------------+  +---------------------------+
  | 케이스 A          |  | tqqq_shares > 0 ?          |
  | 최초 포지션       |  +---------------------------+
  | 즉시 전량매도     |   |YES                    |NO
  +------------------+   v                       v
           |     +--------------------+  +--------------------+
           |     | trigger_type == ?   |  | trigger_type == ?   |
           |     +--------------------+  +--------------------+
           |      |""    |"ma200" |"selloff"  |"selloff"  |"ma200"
           |      v      v        v           v            v
           |   [B분기] [케이스C] [케이스D]   [케이스E]   [케이스F]
           |      |
           |      v
           | +------------------------+
           | | rally_level >= 1 ?      |
           | +------------------------+
           |  |YES            |NO
           |  v               v
           | [B-1]    +------------------+
           |          | sell_off_count   |
           |          | >= 1 ?           |
           |          +------------------+
           |           |YES        |NO
           |           v           v
           |         [B-2]       [B-3]
           v
```

### 각 케이스 상태 전이 상세

```
  +-----------------------------------------------------------------+
  | 케이스 A: initial_pos == True (최초 포지션)                       |
  +-----------------------------------------------------------------+
  | 진입조건: initial_pos=True, 첫 -3% 발생                          |
  | 동작:                                                            |
  |   ath_qqq = max(ath_qqq, running_max_qqq)                       |
  |   SELL_ALL → tqqq_shares = 0                                    |
  |   sell_off_count += 1 (-> 1)                                    |
  |   initial_pos = False                                           |
  |   trigger_type = "selloff"                                      |
  |   waiting = True                                                |
  |   count(1) < heavy_sell_count(4) → 21일 관망                     |
  |     wait_end_idx = i + 21                                       |
  |     split_buy_base = 0.0 (분할매수 없음)                          |
  |   rally_level = 0                                               |
  |   헷지 매수                                                      |
  |                                                                 |
  | 전이: 초기포지션 → [S1] (selloff관망, tqqq=0, count<4)           |
  | 탈출: 21일 후 재진입 (tqqq_shares==0 확인 → 통과)                 |
  | 데드락: 없음 (분할매수 비활성, tqqq=0 보장)                        |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 B-1: trigger_type="", rally_level >= 1                    |
  +-----------------------------------------------------------------+
  | 진입조건: 정상보유 중, 랠리 1단계 이상, -3% 발생                   |
  | 동작: SELL_ALL → tqqq=0                                         |
  |   sell_off_count에 따라 21일 또는 42일 관망                       |
  | 전이: 공격모드 → [S1] 또는 [S2]                                  |
  | 데드락: 없음 (1~3차면 분할매수 없음, 4차+면 42일 후 재진입)         |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 B-2: trigger_type="", rally_level==0, sell_off_count >= 1  |
  +-----------------------------------------------------------------+
  | 진입조건: 정상보유 중, 랠리 없음, 2차 이상 -3%                     |
  | 동작: SELL_ALL → tqqq=0                                         |
  | 전이: 공격모드 → [S1] 또는 [S2]                                  |
  | 데드락: 없음 (위와 동일)                                          |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 B-3: trigger_type="", rally_level==0, sell_off_count == 0  |
  +-----------------------------------------------------------------+
  | 진입조건: 재진입 직후, 랠리 없음, 첫 -3%                          |
  | 동작: 무시 (매도하지 않음)                                        |
  |   sell_off_count += 1 (-> 1)                                    |
  |   event_handled = True                                          |
  | 전이: 공격모드 유지 (변경 없음)                                    |
  | 데드락: 없음 (다음 -3%는 B-2로 전량매도됨)                         |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 C: tqqq_shares > 0, trigger_type == "ma200"               |
  +-----------------------------------------------------------------+
  | 진입조건: MA200 관망 중 분할매수 포지션 있음 + -3% 발생            |
  | 동작: SELL_ALL (분할매수 포지션 포함 전량 청산!)                    |
  |   trigger_type = "selloff" (MA200에서 selloff로 전환)            |
  |   sell_off_count에 따라 21일 또는 42일 관망                       |
  | 전이: [M2] → [S1] 또는 [S2]                                     |
  | 데드락: 없음 (전량매도로 tqqq=0 보장)                              |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 D: tqqq_shares > 0, trigger_type == "selloff"             |
  +-----------------------------------------------------------------+
  | 진입조건: selloff 관망 중 분할매수 포지션 있음 + -3% 재발동         |
  | 동작: 매도 안 함! 분할매수 포지션 유지!                             |
  |   sell_off_count += 1                                           |
  |   count < 4: wait_end_idx 리셋 (21일 다시 시작)                  |
  |   count >= 4: 42일 타이머 리셋 + split_buy_base 갱신             |
  | 전이: [S3] → [S3] 또는 [S4]                                     |
  |                                                                 |
  | *** 핵심 데드락 시나리오 ***                                      |
  | count < 4에서 -3% 반복 발생 시:                                   |
  |   wait_end_idx만 계속 리셋됨                                     |
  |   tqqq_shares > 0 유지 (분할매수 포지션)                          |
  |   하지만 재진입 체크는 tqqq_shares==0 요구                        |
  |   → 21일 타이머가 만료되어도 재진입 불가!                          |
  |   → 다른 -3%가 안 오면 21일 후 재진입 시도하지만                   |
  |     tqqq_shares > 0이라 건너뜀                                   |
  |   → [데드락 상태 1] 발생 (아래 상세 분석 참조)                      |
  |                                                                 |
  | count >= 4로 전환 시:                                             |
  |   42일 타이머 시작 + split_buy_base 갱신                          |
  |   → 추가 분할매수로 tqqq_shares 더 증가 가능                      |
  |   → 42일 후 재진입 체크에서도 tqqq_shares > 0이면 실패            |
  |   → [데드락 상태 2] 발생                                         |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 E: tqqq_shares == 0, trigger_type == "selloff"            |
  +-----------------------------------------------------------------+
  | 진입조건: selloff 관망 중 포지션 없음 + -3% 재발동                 |
  | 동작: 케이스 D와 동일 (카운트/타이머 업데이트만)                    |
  | 전이: [S1] → [S1] 또는 [S2]                                     |
  | 데드락: 없음 (tqqq=0이므로 타이머 만료 시 재진입 가능)              |
  +-----------------------------------------------------------------+

  +-----------------------------------------------------------------+
  | 케이스 F: tqqq_shares == 0, trigger_type == "ma200"              |
  +-----------------------------------------------------------------+
  | 진입조건: MA200 관망 중 포지션 없음 + -3% 발동                     |
  | 동작: timer_42_start_idx = i (타이머 갱신만)                      |
  |   sell_off_count 증가 없음!                                      |
  | 전이: [M1] 유지                                                  |
  | 데드락: 없음 (MA5>MA200 streak 조건 충족 시 재진입)                |
  +-----------------------------------------------------------------+
```

## 4. 이벤트 2: MA200 터치

```
  +=============================================+
  |         MA200 터치 이벤트 체크                |
  +=============================================+
                      |
          +---------------------------+
          | event_handled == False     |
          | AND use_ma200_sell == True  |
          | AND rally_level > 0        |
          | AND tqqq_shares > 0        |
          | AND q <= ma200 * 1.01      |
          +---------------------------+
           |YES                    |NO
           v                       v
  +--------------------+    [다음 이벤트로]
  | MA200 전량매도      |
  |                    |
  | SELL_ALL → tqqq=0  |
  | trigger_type="ma200"|
  | waiting=True       |
  | split_buy_base = q |
  | ma5_streak = 0     |
  | 헷지 매수           |
  +--------------------+
           |
           v
  전이: 공격모드 → [M1] (ma200관망, tqqq=0)

  탈출 경로:
    MA5 > MA200 연속 5일 → 재진입 (tqqq==0 확인 필요)
    분할매수 발동 시 → [M2]로 전이 (tqqq>0)
      이후 -3% 오면 → 케이스 C (전량매도 → [S1]/[S2])
      -3% 안 오면 → MA5>MA200 streak 충족해도 재진입 불가!
        → [데드락 상태 3] (아래 분석 참조)

  데드락: 분할매수로 tqqq>0 되면 재진입 불가 (데드락 상태 3)
```

## 6. 이벤트 4: 분할매수

```
  +=============================================+
  |         분할매수 체크                          |
  +=============================================+
  | 조건: waiting==True AND split_buy_base > 0    |
  | (event_handled 플래그와 무관하게 항상 실행!)     |
  +=============================================+
                      |
                      v
  +----------------------------------------------+
  | 각 레벨(A1~A5, B1~B5) 순회:                    |
  |   이미 매수한 레벨? → skip                      |
  |   B구간 && vix_b_unlocked==False? → skip       |
  |   q <= split_buy_base * (1-drop_pct)? → 매수   |
  +----------------------------------------------+
                      |
                      v
  매수 시:
  +----------------------------------------------+
  | hedge_val + cash = total_liquid (매 레벨 재계산) |
  | amount = total_liquid * buy_pct (10%)          |
  | 현금 부족 → 헷지 일부 매도                       |
  | TQQQ 매수 → tqqq_shares += shares_bought      |
  +----------------------------------------------+
                      |
                      v
  *** 상태 변화 ***
  tqqq_shares: 0 → > 0  (분할매수 체결)
  waiting: True 유지
  trigger_type: 변경 없음 ("selloff" 또는 "ma200")
  split_buy_base: 변경 없음

  → 이 시점에서 재진입 경로가 차단됨!
    (tqqq_shares > 0이므로 line 438 조건 실패)
```

### 분할매수 활성화 조건 정리

```
  +------------------------------------------+
  | 케이스별 split_buy_base 설정               |
  +------------------------------------------+
  | sell_off_count 1~3 (21일 관망):            |
  |   split_buy_base = 0.0                    |
  |   → 분할매수 비활성 → tqqq=0 유지           |
  |   → 재진입 가능 (데드락 없음)               |
  +------------------------------------------+
  | sell_off_count >= 4 (42일 관망):            |
  |   split_buy_base = q (현재가)              |
  |   → 분할매수 활성 → tqqq>0 가능            |
  |   → 재진입 불가능 가능성 (데드락!)           |
  +------------------------------------------+
  | MA200 관망:                                |
  |   split_buy_base = q (항상)                |
  |   → 분할매수 활성 → tqqq>0 가능            |
  |   → 재진입 불가능 가능성 (데드락!)           |
  +------------------------------------------+
```

## 7. 이벤트 5: 랠리 익절

```
  +=============================================+
  |         랠리 익절 체크                         |
  +=============================================+
  | 조건: event_handled==False                    |
  |       AND tqqq_shares > 0                    |
  |       AND trigger_type == ""                  |
  +=============================================+
                      |
                      v
  gain = (q - ath_qqq) / ath_qqq

  +-- gain >= +9% AND rally_level < 1 → 10% 매도, rally_level=1
  |
  +-- gain >= +21% AND rally_level < 2 → 20% 매도, rally_level=2
  |
  +-- gain >= +28% AND rally_level < 3 → 30% 매도, rally_level=3

  주의: trigger_type=="" 조건이므로 관망 중(waiting=True)에는 절대 발동 안 함
        분할매수로 tqqq>0이어도 trigger_type이 "selloff"/"ma200"이면 발동 안 함
```

## 8. 데드락 상태 상세 분석

### 데드락 상태 1: selloff 관망 + 분할매수 + count < 4

```
  발생 시나리오:
  ================================================================
  1. 4차+ -3% 전량매도 → 42일 관망 시작, split_buy_base = q
  2. QQQ가 하락하여 분할매수 체결 → tqqq_shares > 0
  3. 42일 타이머 만료
  4. 재진입 체크: tqqq_shares == 0? → NO → 건너뜀!
  5. 다음날도, 그 다음날도... 영원히 재진입 불가

  탈출 경로 분석:
  ================================================================
  (a) -3% 재발동 → 케이스 D
      → 매도 안 함! 포지션 유지!
      → sell_off_count 증가 + 타이머 리셋
      → 다시 42일 기다려야 함 → 다시 데드락

  (b) MA200 터치 → 조건: rally_level > 0 필요
      → 관망 중에는 trigger_type="selloff"이므로
        랠리 익절 자체가 발동 안 됨 (trigger_type="" 필요)
      → rally_level은 항상 0
      → MA200 터치 조건 불충족 → 이 경로 없음

  (c) 자연적 재진입 → tqqq_shares == 0 필요
      → 분할매수 포지션을 매도할 메커니즘이 없음!
      → 이 경로 없음

  결론: 완전한 데드락. 탈출 불가능.
  ================================================================

  상태 전이 사이클 (무한 루프):
  [S4: selloff관망, tqqq>0, count>=4]
     → 42일 경과 → 재진입 시도 → tqqq>0 → 실패
     → -3% 발생 → 케이스 D → 타이머 리셋 → [S4]
     → 42일 경과 → 재진입 시도 → tqqq>0 → 실패
     → ... (무한 반복)
```

```
    -3%가 오면 → 케이스 C로 탈출 가능 (전량매도 후 selloff 관망으로 전환)
```

```
    단, 불필요한 상태 변경이므로 혼동의 여지 있음
```

```
  상황: use_trend_filter=True, trend_max_wait=0 (무기한)
```
